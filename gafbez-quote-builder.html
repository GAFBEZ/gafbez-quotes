🔧 One-Time Setup Guide

Follow these steps once. After that you just share the URL and everyone uses the same live data.

Step 1 — Create a free Firebase project

  1. Go to console.firebase.google.com and sign in with a Google account.
  2. Click Add project → name it (e.g. gafbez-quotes) → disable Google Analytics → click Create project.
  3. Once created, click the Web icon (</>) on the project home screen.
  4. Name the app (e.g. quote-builder) → click Register app.
  5. Copy the entire firebaseConfig object shown (the one with apiKey, databaseURL, etc.).

Step 2 — Enable Realtime Database

  1. In the left sidebar click Build → Realtime Database.
  2. Click Create Database → choose a region → select Start in test mode → Enable.
  3. Copy the database URL (looks like https://your-project-default-rtdb.firebaseio.com) — you need it in the config.

Step 3 — Paste the config into this file

  1. Open gafbez-quote-builder.html in Notepad (or any text editor).
  2. Find the section that says PASTE YOUR FIREBASE CONFIG HERE.
  3. Replace the placeholder firebaseConfig = { ... } block with the one you copied from Firebase.
  4. Save the file.

Step 4 — Set your PINs

  1. In the same file, find the APP_PINS section in the JavaScript below.
  2. Change admin: "1234" to your chosen admin PIN (only you use this).
  3. Change team: "5678" to your chosen team PIN (your secretaries use this).
  4. Save the file again.

Step 5 — Host for free on GitHub Pages

  1. Create a free account at github.com.
  2. Click + → New repository → name it e.g. gafbez-quotes → Public → Create.
  3. Click Add file → Upload files → drag your HTML file → Commit changes.
  4. Go to Settings → Pages → Source: Deploy from a branch → Branch: main → folder: / (root) → Save.
  5. After ~60 seconds your URL will appear, e.g. https://yourusername.github.io/gafbez-quotes/gafbez-quote-builder.html
  6. Share that URL with your secretaries. Done!

How it works day-to-day

  • 🔑 You (Admin PIN) — can edit product names, descriptions, prices, payment details, and T&Cs. Click "Save Catalogue to Cloud" and all secretaries see the update within seconds.
  • 👩‍💼 Secretaries (Team PIN) — see the latest catalogue automatically. They fill in customer details, pick items, set quantities, and print. They cannot change the catalogue.
  • 🖨️ Printing — the quote stays local to each session (customer name, quote number etc.). Only shared data (catalogue, company info, T&Cs) syncs in real-time.

✅ Firebase free tier (Spark plan) gives you 1 GB storage, 10 GB/month downloads, and simultaneous connections — more than enough for a small team.
